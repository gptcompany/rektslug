# Implementation Plan: Adaptive Observation Layer

**Branch**: `042-adaptive-observation-layer` | **Date**: 2026-05-01 | **Spec**: [spec.md](spec.md)

## Summary

Replace all hardcoded observation parameters in the spec-041 scorecard with
data-derived functions. Touch tolerance becomes a function of local realized
volatility. Observation windows close on volume-clock, not wall-clock. Bucket
boundaries are empirical quantiles. Dominance comparisons use bootstrap CI.
Regime labels are inferred from volatility quantiles. Zero fixed market
thresholds, bands, or bucket boundaries remain in the adaptive observation
layer; only documented computation-method constants remain.

## Technology Choices

| Choice | Justification |
|--------|---------------|
| stdlib `random` for bootstrap | No dependency; deterministic with seed; 1000 iters on <10k obs is fast |
| stdlib `math.log` for vol | Single function, no numpy needed |
| stdlib `statistics.stdev` for rolling std | Built-in, correct, no dependency |
| Quantile from existing `_compute_quantiles()` | Already tested and production-ready |
| `enable_adaptive` flag | Backward compatible; no breaking change |

## Risk Mitigations

| Risk | Impact | Mitigation |
|------|--------|------------|
| Bootstrap slows pipeline | >10s per run | Profile; if slow, reduce to 500 iters or parallelize |
| Volume data missing in price path | Volume-clock unusable | FR-015: mark as unavailable, time-based as secondary |
| Too few observations for quantile buckets | Empty buckets | FR-006: fall back to single "all" bucket |
| Volatility estimate unstable for small windows | Noisy touch band | Use minimum 20 ticks for vol estimate; flag if fewer |
| Regime inference produces one regime for all data | Useless slicing | Acceptable — means market was stable; "unknown" fallback |

Foundational volatility primitive rules:
- normalize tick timestamps through the same accepted shapes as the scorecard builder
- sort the input path by timestamp before lookback slicing
- accept ISO timestamps and epoch values, not only pre-normalized datetimes
- freeze adaptive touch-band MVP heuristics explicitly:
  - `max(1, floor(local_volatility_bps / 500))` for volatility-derived bands
  - `max(1, floor(spread_bps / 2))` for spread-proxy fallback

## Phase Breakdown

### Phase 1: Adaptive Primitives (`adaptive.py`)

Build the foundational computation functions:
1. `compute_realized_volatility()` — rolling log-return std
2. `compute_adaptive_touch_band()` — vol → touch band bps
3. `compute_quantile_buckets()` — empirical quantile boundaries
4. `infer_regime_map()` — vol quantile → regime labels
5. `compute_volume_threshold()` — volume-clock window threshold

### Phase 2: Bootstrap Dominance (`bootstrap.py`)

1. `bootstrap_dominance()` — pairwise metric comparison with CI
2. Deterministic seeding per slice_id for reproducibility

### Phase 3: Model Extensions (`models/scorecard.py`)

Extend existing Pydantic models with new optional fields:
- `ExpertSignalObservation`: adaptive_touch_band_bps, local_volatility_bps,
  volume_at_touch, volume_window_complete, post_touch_volume, inferred_regime
- `ExpertScorecardSlice`: bucket_boundaries
- `ExpertScorecardBundle`: adaptive_parameters

### Phase 4: Builder Integration (`builder.py`)

1. Extend `apply_touch_detection()` to accept adaptive band
2. Extend `apply_post_touch_path()` to use volume-clock
3. Preserve non-adaptive codepath when `enable_adaptive=False`

### Phase 5: Slicer Integration (`slicer.py`)

1. Replace `_get_distance_bucket()` with quantile-based bucketing
2. Replace `_get_confidence_bucket()` with quantile-based bucketing
3. Replace manual `regime_map` with inferred regime
4. Preserve non-adaptive codepath

### Phase 6: Pipeline Wiring (`pipeline.py`)

1. Add `enable_adaptive` parameter to `run()` and `run_from_retained_snapshots()`
2. Wire adaptive primitives, bootstrap dominance, quantile buckets
3. Include `adaptive_parameters` in bundle output
4. End-to-end test with `enable_adaptive=True`
